## 一、文档概述

### 1.1 产品背景

18–28 岁年轻人常在导师沟通、实习职场、社团协作、正式办事、陌生社交中卡在"我该怎么说"。现有方案（AI 对话、搜索话术模板）存在答案太泛、太正式、不像自己会说的话等问题，且需要反复补充背景信息（"我是学生、对方是导师、我想委婉催一下"）。

### 1.2 产品目标

打造一款**面向不同场景、不同对象，用不同风格语言表达的移动端工具**。通过前置"场景+对象+风格"的可选择卡片，让表达生成更贴近日常使用，帮助用户把"话到嘴边"的犹豫时刻转化为可发送、可开口的表达版本。

### 1.3 产品定位

- **不是**通用 AI 写作工具
- **不是**情感陪伴/聊天机器人
- **是**聚焦"开口前那一刻"的场景表达转换器：把用户心里想说的"大白话/情绪话"，转换成适合特定场景、特定对象、特定风格的可发送文本

### 1.4 一句话定义

**话到嘴边**是一款面向年轻人的场景表达转换器。在校园、职场、社交和正式场景中，用户选择场景、对象、风格，输入真实想法，系统生成微信/IM、邮件/书面、当面沟通三个可直接使用的表达版本。

### 1.5 核心公式

```
表达结果 = 场景 × 对象 × 风格 + 真实想法 + 语气参数
```

## 二、用户分析

### 2.1 目标用户画像

| 维度 | 描述 |
|------|------|
| 年龄 | 18–28 岁 |
| 身份 | 大学生、研究生、实习生、职场 1–3 年新人 |
| 核心痛点 | 有表达意愿，但缺乏场景化表达经验；对导师、领导、正式机构有"不敢直接说"的心理门槛 |
| 使用习惯 | 习惯用微信/IM 沟通，偶尔需要邮件或当面沟通；希望表达"像自己会说的话" |
| 触发场景 | 导师突然问论文进度、职场被领导问项目情况、活动现场遇到想请教的前辈、需要拒绝朋友但不想撕破脸 |

### 2.2 用户核心场景

| 场景 | 对象 | 用户原话 | 转换目标 |
|------|------|----------|----------|
| 校园 | 导师 | "论文还没开始，别催了" | 委婉说明进度，争取时间 |
| 校园 | 同学 | "小组作业有人没干活，但我不想撕破脸" | 体面提醒，推动协作 |
| 职场 | 领导 | "这活不是我负责的，别老找我" | 清晰划清边界，不撕破脸 |
| 职场 | 同事 | "你上次说的数据有问题，我不想显得在挑刺" | 委婉指出问题，保留关系 |
| 社交 | 朋友 | "我不想参加这次聚会" | 体面拒绝，保留关系 |
| 正式事务 | 客服/机构 | "你们办事效率太低了" | 正式投诉/催促，有理有据 |

### 2.3 用户使用路径

```
触发时刻：遇到需要开口但不知道怎么说的场景
  ↓
打开产品 → 首页 → 进入输入页
  ↓
选择场景 → 选择对象 → 选择风格 → 输入真实想法 → 调节语气
  ↓
获得 3 个版本（微信/邮件/当面）→ 复制/分享 → 发送/开口
```

```

├── 首页（Home）
│   ├── 主视觉区（品牌展示）
│   ├── 核心文案区
│   ├── 最近使用（历史记录）
│   ├── 热门风格（快捷入口）
│   └── 核心CTA（进入输入页）
├── 输入页（Compose）
│   ├── 场景选择（4 大场景）

│   ├── 真实想法输入框

├── 语气仪表盘（Tone）
│   ├── 实时预览区

│   ├── 三版本结果卡片（微信/邮件/当面）

│   └── 收藏管理
└── 我的页（Profile）
    ├── 个人偏好
    ├── 使用统计
    └── 反馈入口
```

```
  ↓

  ↑        ↑                                    ↑
```

### 3.3 页面跳转逻辑

| 从页面 | 操作 | 到页面 | 携带数据 | 备注 |
|--------|------|--------|----------|------|
| 首页 | 点击"开始转换" | 输入页 | 无 | 全新会话 |
| 首页 | 点击热门风格 | 输入页 | preset_style | 预填充风格，场景默认"上次使用"或"workplace" |
| 首页 | 点击最近使用 | 结果页 | history_id | 展示历史结果 |

| 结果页 | 点击"再润色" | 语气页 | 保留当前参数 | — |
| 结果页 | 点击"换风格" | 结果页（刷新） | 新 style | 仅当前卡片更新 |
| 结果页 | 点击"换一种风格" | 结果页（刷新） | 新 style | 全部卡片更新 |
| 任意页 | 点击底部导航 | 对应页面 | — | 选中态高亮 |

### 5.2 输入页（Compose）—— 重构核心页

#### 5.2.1 需求说明

**用户场景**：用户进入输入页，依次完成"场景选择 → 对象选择 → 风格选择 → 输入真实想法"，然后进入语气微调。

**前置条件**：从首页 CTA 或热门风格入口进入

**功能范围**：前端 UI、级联选择逻辑、输入校验、数据传递

**核心设计原则**：
- 场景、对象、风格、输入四项为**必填项**，全部完成后才可点击"开始转换"
- 场景选择后动态展开对象选项
- 风格选择后显示风格说明
- 输入框提供场景相关的示例文本

```typescript
interface AppState {
  // 场景选择

  timestamp: number;            // 时间戳
  scene: string;
```

## 八、异常与边界处理

### 8.1 异常处理矩阵

| 场景 | 触发条件 | 前端处理 | 用户感知 | 数据影响 |
|------|----------|----------|----------|----------|
| 网络异常 | 请求超时/断网 | 使用本地 fallback 文本（预设模板） | 静默使用，不暴露技术提示 | 无 |
| API 超时（>5s） | 请求超过 5s 未响应 | 超时后使用 fallback | 显示"正在努力组织语言…"，超时后展示结果 | 无 |
| 输入为空 | 点击"开始转换" | 前端拦截，按钮禁用 | 提示"先输入至少 2 个字" | 无 |
| 输入超限（>300字） | 输入超过 300 字 | 软限制，允许输入但提示 | 字数统计变红"301/300" | 可继续输入至 500 字 |
| 输入硬限制（>500字） | 输入超过 500 字 | 硬限制，截断输入 | 无法继续输入 | 截断至 500 字 |
| 未选场景/对象/风格 | 点击"开始转换" | 前端拦截，按钮禁用 | 提示"请先选择场景、对象和风格" | 无 |
| 生成失败 | API 返回错误 | 显示 fallback + "重试"按钮 | "生成遇到了一点问题，试试重试？" | 无 |
| 快速连续点击 | 500ms 内多次点击 | 防抖 500ms + loading 态 | 按钮显示 loading 动画 | 仅发送一次请求 |
| 无历史记录 | 历史页无数据 | 显示空状态引导 | "还没有开口记录，去首页试试？" | 无 |
| 历史记录过期 | 点击已删除的历史 | 提示记录不存在 | "记录已过期" | 从列表移除 |
| 复制失败 | Clipboard API 失败 | 降级为 execCommand 或提示手动复制 | "复制失败，请手动复制" | 无 |
| 分享失败 | Web Share API 不支持 | 降级为复制链接 | 无提示（静默降级） | 无 |

### 8.2 Fallback 策略

当 API 不可用或超时时，使用前端预设模板生成结果。Fallback 模板按 scene+target+style 组合预置，确保基本可用性。

**Fallback 模板示例**（职场-领导-别甩给我）：
- 微信版："领导，关于这个任务，我想确认一下职责分工。目前看这块不在我负责范围内，建议我们先对齐一下？"
- 邮件版："您好，关于此事，我希望明确职责边界。经核实，该任务不在我的职责范围内，建议确认负责人后再推进。"
- 当面版："领导，我想说一下这个任务的事。我的理解是这块不是我负责的，咱们确认一下？"

**Fallback 不可暴露给用户**：不显示"已切到演示兜底"等技术提示，静默使用。

### 12.2 风格定义详细说明

| 风格ID | 风格名称 | 图标 | 核心策略 | 典型表达特征 |
